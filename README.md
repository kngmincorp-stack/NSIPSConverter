# NSIPS TXT → Excel 転記ツール

レセコン(NSIPS)が出力する TXT ファイルを監視し、新しいTXTが出来たら
内容を抽出して Excel ひな形に転記、**元TXTと同名の .xlsx** として
出力フォルダへ自動保存する常駐GUIツールです。

## 別PCでの使い方（exe版）
1. `dist\NSIPSConverter.exe` を別PCの任意のフォルダにコピー。
2. ダブルクリックで起動（Python不要・単一ファイルで動作）。
3. 画面で**監視元フォルダ**と**出力先フォルダ**を「参照…」で指定。
4. 「監視開始」で自動監視スタート。以後 `NSIPSConverter.exe` を置いておくだけ。
   - 設定は同じフォルダの `config.json` に自動保存されます。

## 画面の操作
- **監視開始／停止**：3秒間隔でフォルダをチェックし、新規TXTを自動Excel化。
- **今すぐ全処理**：未処理TXTをその場でまとめて処理。
- **処理済み履歴クリア**：全TXTを再処理したいとき。
- **開く**：各フォルダをエクスプローラーで開く。
- **更新履歴**：これまでの変更内容(CHANGELOG)を表示。
- **更新を確認**：手動で最新版チェック。

## パッチ更新システム
- 起動時に GitHub Releases を確認し、新しいバージョンがあれば自動でお知らせ→
  確認後にダウンロードして自身を置き換え、再起動します（exe版のみ）。
- 更新後の初回起動で「更新履歴」が自動表示されます。
- 配布リポジトリ: `kngmincorp-stack/NSIPSConverter`（public）

### 開発者向け：新パッチの公開手順
1. コードを修正。
2. `CHANGELOG.md` の先頭に新バージョン項目を**事務員向けの平易な日本語**で追記。
3. PowerShell で公開（VERSION更新→ビルド→Release作成を自動実行）:
   ```powershell
   .\publish_patch.ps1 -Version 1.0.1
   ```
   - `python build_exe.py` で単体ビルドのみも可能（生成物 `dist\NSIPSConverter.exe`）。

## 転記マッピング（TXTの「1,」行＝患者情報）
| Excel列 | 項目 | 抽出元 | 変換ルール |
|---|---|---|---|
| A | 患者番号 | 2番目 | そのまま |
| B | カナ氏名 | 3番目 | そのまま |
| C | 漢字氏名 | 4番目 | そのまま |
| D | 生年月日 | 6番目 | 8桁→`YYYY/MM/DD` |
| E | 性別 | 5番目 | 1→男 / 2→女 |
| F | 保険者番号 | 住所等を飛ばした位置 | そのまま |
| G | 記号 | 保険者番号の隣 | そのまま |
| H | 番号 | 記号の隣 | そのまま |
| I | 枝番 | 行末の値 | そのまま |
| J | 本/家 | 番号の隣 | 1→本人 / 2→家族 |
| K | 最終来局日 | 「2,」行の日付(最新) | 8桁→`YYYY/MM/DD` |

- 住所・郵便番号は出力しません（保険者番号まで飛ばします）。
- 生年月日はTXTの8桁数値を正としてスラッシュ整形します。

## 構成ファイル
- `nsips_converter.py` … 本体（GUI・解析・自動更新）
- `template.xlsx` … Excelひな形（患者データなし）※消さないでください
- `VERSION` / `CHANGELOG.md` … バージョン・更新履歴（exeに同梱）
- `build_exe.py` … exeビルド（PyInstaller onefile）
- `publish_patch.ps1` … パッチ公開（PowerShell）
- `起動.bat` … ソース版の起動用
- `config.json` … 設定（自動生成／gitignore対象）

## 必要環境
- exe版：不要（単一実行ファイル）
- ソース版：Python 3 + `openpyxl`（`pip install openpyxl`）
