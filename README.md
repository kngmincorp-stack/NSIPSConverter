# NSIPS TXT → CSV 転記ツール

レセコン(NSIPS)が出力する TXT ファイルを監視し、新しいTXTが出来たら
内容を抽出して、**元TXTと同名の .csv**（Shift-JIS・見出し付き）として
出力フォルダへ自動保存する常駐GUIツールです。

## 別PCでの使い方（exe版）
1. **`配布\NSIPSConverter.exe`** を別PCの任意フォルダにコピー（このexe1つだけでOK。Python不要）。
2. ダブルクリックで起動。
3. 画面で**監視元フォルダ**（NSIPSのTXT出力先）と**出力先フォルダ**（CSV保存先）を「参照…」で指定。
4. 「監視開始」で自動監視スタート。以降はこのexeを置いておくだけ。
   - 設定は同じフォルダの `config.json` に自動保存。
   - **一度フォルダを設定すれば、次回起動時は自動で監視を再開**します。

## 画面の操作
- **監視開始／停止**：3秒間隔でフォルダをチェックし、新規TXTを自動CSV化。
- **今すぐ全処理**：未処理TXTをその場でまとめて処理（初回の全件処理に）。
- **処理済み履歴クリア**：全TXTを再処理したいとき。
- **Windowsスタートアップに登録**（チェックボックス）：PC起動時に自動で立ち上げ。
- **開く**：各フォルダをエクスプローラーで開く。
- **更新履歴 / 更新を確認**：変更内容の表示・最新版チェック。

## 処理済みTXTの扱い
- 一度処理したTXTは**ファイル名**で記録します（`config.json`）。
- 最初に監視を始めたときは、フォルダ内の**全TXTを処理**。
- その後の**監視再開・アプリ再起動でも、処理済みTXTは再処理しません**（新しい分だけ処理）。

## 出力CSV
- 文字コード：**Shift-JIS (cp932)** ＝ Excelでそのまま開いても文字化けしません。
- 1行目：見出し（患者番号, カナ氏名, 漢字氏名, 生年月日, 性別, 保険者番号, 記号, 番号, 枝番, 本/家, 最終来局日）
- 2行目：データ。1つのTXT＝1つのCSV。

## パッチ更新システム
- 起動時に GitHub Releases を確認し、新しいバージョンがあれば自動でお知らせ→
  確認後にダウンロードして自身を置き換え、再起動します（exe版のみ）。
- 更新後の初回起動で「更新履歴」が自動表示されます。
- 配布リポジトリ: `kngmincorp-stack/NSIPSConverter`（public）

### 開発者向け：新パッチの公開手順
1. コードを修正。
2. `CHANGELOG.md` の先頭に新バージョン項目を平易な日本語で追記。
3. PowerShell で公開（VERSION更新→ビルド→Release作成を自動実行）:
   ```powershell
   .\publish_patch.ps1 -Version 1.1.1
   ```
   - `python build_exe.py` で単体ビルドのみも可能（生成物 `配布\NSIPSConverter.exe`）。

## 転記マッピング（TXTの「1,」行＝患者情報）
| 項目 | 抽出元 | 変換ルール |
|---|---|---|
| 患者番号 | 2番目 | そのまま |
| カナ氏名 | 3番目 | そのまま |
| 漢字氏名 | 4番目 | そのまま |
| 生年月日 | 6番目 | 8桁→`YYYY/MM/DD` |
| 性別 | 5番目 | 1→男 / 2→女 |
| 保険者番号 | 住所等を飛ばした位置 | そのまま |
| 記号 | 保険者番号の隣 | そのまま |
| 番号 | 記号の隣 | そのまま |
| 枝番 | 行末の値 | そのまま |
| 本/家 | 番号の隣 | 1→本人 / 2→家族 |
| 最終来局日 | 「2,」行の日付(最新) | 8桁→`YYYY/MM/DD` |

- 住所・郵便番号は出力しません（保険者番号まで飛ばします）。

## 構成ファイル
- `nsips_converter.py` … 本体（GUI・解析・CSV出力・自動更新）
- `VERSION` / `CHANGELOG.md` … バージョン・更新履歴（exeに同梱）
- `build_exe.py` … exeビルド（PyInstaller onefile）
- `publish_patch.ps1` … パッチ公開（PowerShell）
- `配布\NSIPSConverter.exe` … 別PC配布用の単一実行ファイル（ビルドで生成）
- `config.json` … 設定（自動生成／gitignore対象）

## 必要環境
- exe版：不要（単一実行ファイル・Windows）
- ソース版：Python 3（標準ライブラリのみ。openpyxl不要）
